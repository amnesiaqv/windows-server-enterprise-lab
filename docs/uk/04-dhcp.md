# 📡 Конфігурація DHCP-сервера

## Огляд

У цьому документі описано процес розгортання та налаштування служби **Dynamic Host Configuration Protocol (DHCP)** у лабораторному середовищі **Enterprise Active Directory Lab**.

DHCP-сервер автоматично призначає IPv4-адреси, налаштування DNS та інші мережеві параметри клієнтським комп’ютерам, приєднаним до домену. Це спрощує адміністрування мережі та забезпечує однакову конфігурацію всіх пристроїв.

---

## Роль DHCP-сервера

Роль **DHCP Server** була встановлена на контролері домену (**DC01**).

| Параметр           | Значення                         |
| ------------------ | -------------------------------- |
| DHCP-сервер        | DC01                             |
| Операційна система | Windows Server 2025              |
| Авторизація        | Авторизований в Active Directory |
| Сімейство адрес    | IPv4                             |

Перед початком видачі IP-адрес клієнтам DHCP-сервер був авторизований в Active Directory.

---

## DHCP Scope

Для внутрішньої лабораторної мережі була налаштована одна область IPv4.

| Налаштування          | Значення         |
| --------------------- | ---------------- |
| Назва області         | Internal Network |
| Мережа                | 192.168.10.0/24  |
| Початкова адреса      | 192.168.10.100   |
| Кінцева адреса        | 192.168.10.200   |
| Маска підмережі       | 255.255.255.0    |
| Шлюз за замовчуванням | N/A              |
| Бажаний DNS-сервер    | 192.168.10.10    |

Створена область забезпечує автоматичне призначення IP-адрес для всіх клієнтських комп’ютерів Windows 11.

---

## DHCP Options

Було налаштовано такі параметри DHCP.

| Опція                 | Значення         |
| --------------------- | ---------------- |
| DNS Server (006)      | 192.168.10.10    |
| DNS Domain Name (015) | KUZNIETSOV.local |

Ці параметри гарантують, що клієнти отримують необхідні мережеві налаштування під час процесу отримання DHCP-оренди.

---

## DHCP Leases

Після приєднання до домену та оновлення мережевої конфігурації обидва клієнтські комп’ютери успішно отримали IP-адреси від DHCP-сервера.

| Комп’ютер | Джерело адреси |
| --------- | -------------- |
| CLIENT01  | DHCP           |
| CLIENT02  | DHCP           |

Активні оренди відображаються в консолі керування DHCP у розділі **Address Leases**.

---

## Процес отримання DHCP-оренди

Кожен клієнт використовує стандартний процес DHCP:

1. DHCP Discover
2. DHCP Offer
3. DHCP Request
4. DHCP Acknowledgement (ACK)

Після успішного отримання оренди клієнт автоматично отримує:

* IPv4-адресу
* Маску підмережі
* Бажаний DNS-сервер
* DNS-суфікс

## Команди перевірки

Під час тестування використовувалися такі команди.

### Перегляд конфігурації клієнта

```cmd
ipconfig /all
```

### Оновлення DHCP-оренди

```cmd
ipconfig /renew
```

### Звільнення поточної DHCP-оренди

```cmd
ipconfig /release
```

## Підсумок

DHCP-сервер успішно забезпечує автоматичне налаштування IPv4 для всіх доменних клієнтів Windows 11.

Централізоване керування IP-адресами зменшує адміністративне навантаження, мінімізує помилки конфігурації та забезпечує однакові мережеві налаштування в усьому лабораторному середовищі.

Усі клієнти успішно отримали коректні DHCP-оренди, DNS-налаштування та доменну конфігурацію від авторизованого DHCP-сервера.
