# 🏛️ Служби доменів Active Directory

## Огляд

У цьому документі описано процес розгортання та налаштування **Active Directory Domain Services (AD DS)** у лабораторному проєкті **Enterprise Active Directory Lab**.

Active Directory є центральною платформою керування обліковими записами та доступом у лабораторному середовищі. Вона забезпечує централізовану автентифікацію, авторизацію, керування комп'ютерами та розгортання групових політик для всіх пристроїв, приєднаних до домену.

---

## Розгортання Active Directory

Роль **Active Directory Domain Services** була встановлена на віртуальну машину Windows Server 2025 (**DC01**).

Під час розгортання сервер було підвищено до контролера домену, у результаті чого було створено новий ліс і домен Active Directory.

| Параметр | Значення |
| -------- | -------- |
| Контролер домену | DC01 |
| Операційна система | Windows Server 2025 |
| Ім'я домену | KUZNIETSOV.local |
| Функціональний рівень лісу | Windows Server 2025 (за замовчуванням) |
| Функціональний рівень домену | Windows Server 2025 (за замовчуванням) |

---

## Структура організаційних підрозділів

Організаційні підрозділи (Organizational Units, OU) були створені для логічного розділення адміністративних об'єктів і спрощення керування.

Приклад структури:

```text
KUZNIETSOV.local
│
└── Company
    ├── Users
    │   ├── IT
    │   ├── HR
    │   └── Finance
    │
    ├── Groups
    ├── Computers
    └── Servers
```

Розподіл ресурсів між окремими організаційними підрозділами спрощує делегування адміністративних повноважень і застосування об'єктів групової політики (GPO) до певних відділів або типів об'єктів.

---

## Облікові записи користувачів

Доменні облікові записи користувачів були створені для тестування централізованої автентифікації та контролю доступу.

Кожен користувач був розміщений у відповідному організаційному підрозділі та включений до однієї або кількох груп безпеки відповідно до вимог свого підрозділу.

Приклади атрибутів користувача:

* Ім'я користувача (Username)
* Відображуване ім'я (Display Name)
* Відділ (Department)
* Пароль
* Членство в групах

---

## Групи безпеки

Групи безпеки були створені для спрощення керування дозволами.

Замість призначення дозволів окремим користувачам права доступу надаються групам, а користувачі отримують доступ завдяки членству в цих групах.

Типові групи за підрозділами:

* IT
* HR
* Finance

Такий підхід відповідає рекомендованій Microsoft моделі керування доступом на основі ролей (Role-Based Access Control, RBAC).

---

## Комп'ютери домену

Обидві робочі станції під керуванням Windows 11 були успішно приєднані до домену Active Directory.

| Комп'ютер | Статус |
| --------- | :----: |
| CLIENT01 | ✅ Приєднано |
| CLIENT02 | ✅ Приєднано |

Облікові записи комп'ютерів були автоматично створені в Active Directory та керуються за допомогою організаційних підрозділів і групових політик.

---

## Автентифікація

Автентифікація користувачів домену виконується контролером домену.

Основні переваги:

* Централізоване керування обліковими записами
* Єдиний вхід (Single Sign-On, SSO)
* Примусове застосування політик паролів
* Автентифікація на основі Kerberos
* Централізоване адміністрування

Користувачам більше не потрібно створювати окремі локальні облікові записи на кожній робочій станції.

## Адміністративні інструменти

Під час розгортання використовувалися такі консолі керування Microsoft (MMC).

| Інструмент | Призначення |
| ---------- | ----------- |
| Active Directory Users and Computers | Керування користувачами та комп'ютерами |
| Active Directory Administrative Center | Розширене адміністрування |
| Server Manager | Керування ролями сервера |
| Group Policy Management | Керування груповими політиками |

## Перевірка

Розгортання Active Directory було перевірено за допомогою таких тестів.

| Перевірка | Очікуваний результат | Статус |
| ---------- | -------------------- | :----: |
| Контролер домену працює | Успішно | ✅ |
| Домен успішно створено | Успішно | ✅ |
| Автентифікація користувачів | Успішно | ✅ |
| Клієнти приєднані до домену | Успішно | ✅ |
| Об'єкти комп'ютерів відображаються | Успішно | ✅ |
| Групи безпеки створені | Успішно | ✅ |

## Підсумок

Розгорнута інфраструктура Active Directory забезпечує централізоване керування обліковими записами та ідентифікацією в лабораторному середовищі.

Використовуючи організаційні підрозділи, доменні облікові записи користувачів, групи безпеки та клієнтські комп'ютери Windows 11, приєднані до домену, лабораторія демонструє основні принципи корпоративного керування ідентифікацією та адміністративні практики, що широко застосовуються у виробничих середовищах Windows Server.
