# 📁 Конфігурація файлового сервера

## Огляд

У цьому документі описано налаштування ролі **File Server** у лабораторному середовищі **Enterprise Active Directory Lab**.

Контролер домену містить централізовані спільні папки, доступні для авторизованих користувачів домену. Доступ до спільних ресурсів контролюється за допомогою комбінації **Share Permissions**, **NTFS Permissions** та груп безпеки **Active Directory**.

---

## Файловий сервер

Контролер домену на базі **Windows Server 2025** також виконує роль файлового сервера в лабораторному середовищі.

| Параметр           | Значення            |
| ------------------ | ------------------- |
| Сервер             | DC01                |
| Операційна система | Windows Server 2025 |
| Файлова система    | NTFS                |
| Метод доступу      | SMB File Sharing    |

Спільні папки забезпечують централізоване зберігання даних для користувачів, підключених до домену Active Directory.

---

## Спільні папки

Для імітації ресурсів різних відділів було створено кілька спільних папок.

| Ресурс  | Призначення               |
| ------- | ------------------------- |
| Public  | Спільні корпоративні дані |
| IT      | Файли ІТ-відділу          |
| HR      | Документи відділу кадрів  |
| Finance | Фінансові файли           |

---

## Share Permissions

Дозволи спільного доступу (**Share Permissions**) керують мережевим доступом до спільних папок, тоді як дозволи **NTFS** контролюють доступ до файлів і папок на локальній файловій системі.

Дозволи були призначені за допомогою груп безпеки Active Directory, а не окремих облікових записів користувачів.

Такий підхід спрощує адміністрування та відповідає рекомендованим практикам Microsoft.

---

## NTFS Permissions

Дозволи NTFS забезпечують безпеку файлової системи та контроль доступу до файлів і папок, які зберігаються на сервері.

NTFS-дозволи переважно призначалися групам безпеки Active Directory, а не окремим користувачам, що відповідає рекомендованим практикам Microsoft для масштабованого керування доступом.

Використання груп безпеки робить керування дозволами більш гнучким, масштабованим і простішим у підтримці.

---

## Контроль доступу

Доступ до спільних ресурсів побудований за принципом **найменших привілеїв (Principle of Least Privilege)**.

Користувачі отримують лише ті права доступу, які необхідні їм для виконання робочих завдань.

Приклади дозволів:

* Read (Читання)
* Modify (Зміна)
* Full Control (Повний контроль)

Фактичні права доступу визначаються комбінацією **Share Permissions** та **NTFS Permissions**.

---

## Підключення мережевого диска

Мережевий диск був автоматично підключений для користувачів домену за допомогою **Group Policy** або сценарію входу (**logon script**).

Це дозволяє користувачам отримувати доступ до спільних папок без необхідності створювати мережеві підключення вручну.

Приклад:

```text
\\DC01\Public
```

---

## Перевірка

Конфігурацію файлового сервера було перевірено за допомогою таких тестів.

| Тест                                        | Очікуваний результат | Статус |
| ------------------------------------------- | -------------------- | :----: |
| Створено спільні папки                      | Успішно              |    ✅   |
| Налаштовано Share Permissions               | Успішно              |    ✅   |
| Налаштовано NTFS Permissions                | Успішно              |    ✅   |
| Мережевий диск доступний                    | Успішно              |    ✅   |
| Користувачі домену мають доступ до ресурсів | Успішно              |    ✅   |

## Підсумок

Файловий сервер забезпечує централізоване зберігання даних у лабораторному середовищі **Enterprise Active Directory Lab**.

Поєднання SMB file sharing, NTFS-дозволів, Share Permissions та груп безпеки Active Directory демонструє безпечне й масштабоване керування доступом до файлів, яке широко використовується в корпоративних середовищах Windows.
