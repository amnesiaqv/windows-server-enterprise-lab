# 🌐 Конфігурація DNS-сервера

## Огляд

У цьому документі описано процес розгортання та налаштування служби **Domain Name System (DNS)** у лабораторному середовищі **Enterprise Active Directory Lab**.

DNS-сервер інтегрований із **Active Directory Domain Services (AD DS)** та забезпечує розпізнавання імен для всіх комп’ютерів, приєднаних до домену. Це дозволяє клієнтам знаходити контролер домену та інші мережеві ресурси за їхніми іменами хостів замість використання IP-адрес.

---

## Роль DNS-сервера

Роль **DNS Server** була встановлена на контролері домену (**DC01**) під час розгортання Active Directory.

| Параметр           | Значення                       |
| ------------------ | ------------------------------ |
| DNS-сервер         | DC01                           |
| Операційна система | Windows Server 2025            |
| Тип DNS-зони       | Інтегрована з Active Directory |
| Ім’я домену        | KUZNIETSOV.local               |

Служба DNS зберігає DNS-записи в Active Directory та автоматично реплікує їх між контролерами домену, забезпечуючи безпечне й централізоване розпізнавання імен.

---

## Конфігурація DNS-зони

Під час встановлення Active Directory було автоматично створено **Forward Lookup Zone**.

| Зона             | Тип                                       |
| ---------------- | ----------------------------------------- |
| KUZNIETSOV.local | Зона прямого пошуку (Forward Lookup Zone) |

Під час підвищення сервера до ролі контролера домену зона була автоматично налаштована як інтегрована з Active Directory.

---

## Записи хостів

У DNS-зоні були успішно зареєстровані такі записи типу **Host (A)**.

| Ім’я хоста | Призначення           |
| ---------- | --------------------- |
| DC01       | Контролер домену      |
| CLIENT01   | Робоча станція домену |
| CLIENT02   | Робоча станція домену |

Динамічні оновлення DNS дозволяють клієнтським комп’ютерам автоматично реєструвати та оновлювати свої DNS-записи після приєднання до домену.

---

## Процес розпізнавання імен DNS

DNS-служба виконує такі функції:

* Перетворює імена хостів на IP-адреси
* Підтримує автентифікацію Active Directory
* Дозволяє знаходити контролери домену
* Забезпечує взаємодію між членами домену
* Підтримує динамічні DNS-записи

Без правильно налаштованого DNS-сервера вхід до домену та багато служб Active Directory не працювали б належним чином.

---

## Перевірка

Конфігурацію DNS було перевірено за допомогою таких тестів.

| Тест                                 | Очікуваний результат | Статус |
| ------------------------------------ | -------------------- | :----: |
| DNS-сервер працює                    | Успішно              |    ✅   |
| Створено зону прямого пошуку         | Успішно              |    ✅   |
| Записи хостів доступні               | Успішно              |    ✅   |
| Ім’я контролера домену розпізнається | Успішно              |    ✅   |
| Імена клієнтів розпізнаються         | Успішно              |    ✅   |
| Динамічна реєстрація DNS працює      | Успішно              |    ✅   |

---

## Команди перевірки

Для перевірки роботи DNS використовувалися такі команди.

### Перевірка розпізнавання контролера домену

```cmd
nslookup DC01
```

### Перевірка розпізнавання клієнтських комп’ютерів

```cmd
nslookup CLIENT01
```

```cmd
nslookup CLIENT02
```

### Перевірка мережевої конфігурації

```cmd
ipconfig /all
```

На кожному клієнті як бажаний DNS-сервер вказано контролер домену.

## Підсумок

DNS-сервер був успішно інтегрований із **Active Directory Domain Services** та забезпечує централізоване розпізнавання імен у лабораторному середовищі.

Правильна конфігурація DNS забезпечує надійну автентифікацію Active Directory, виявлення мережевих служб, взаємодію клієнтів і коректне функціонування доменної інфраструктури загалом.
