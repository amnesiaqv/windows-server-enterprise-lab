# 🛡️ Конфігурація Group Policy

## Огляд

У цьому документі описано налаштування та розгортання об’єктів групової політики (**Group Policy Objects — GPO**) у лабораторному середовищі **Enterprise Active Directory Lab**.

Group Policy є однією з основних технологій керування в Microsoft Active Directory. Вона дозволяє адміністраторам централізовано налаштовувати параметри операційної системи, застосовувати політики безпеки, автоматизувати адміністративні завдання та стандартизувати робоче середовище користувачів на всіх комп’ютерах, приєднаних до домену.

---

## Керування Group Policy

Об’єкти групової політики були створені та керувалися за допомогою **Group Policy Management Console (GPMC)**.

| Компонент                   | Значення                |
| --------------------------- | ----------------------- |
| Інструмент керування        | Group Policy Management |
| Домен                       | KUZNIETSOV.local        |
| Контролер домену            | DC01                    |
| Операційна система клієнтів | Windows 11 Pro          |

Політики були прив’язані до відповідних організаційних одиниць (**Organizational Units — OUs**), щоб вони застосовувалися лише до визначених користувачів або комп’ютерів.

---

## Реалізовані об’єкти Group Policy

У лабораторному середовищі були успішно налаштовані такі об’єкти групової політики.

| Group Policy           | Опис                                                                   |
| ---------------------- | ---------------------------------------------------------------------- |
| Set Wallpaper          | Налаштовує корпоративні шпалери робочого столу для користувачів домену |
| Hide Local C: Drive    | Приховує локальний диск C: у Провіднику Windows                        |
| Map Network Drives     | Автоматично підключає мережеві диски під час входу користувача         |
| Disable CMD            | Забороняє запуск командного рядка                                      |
| Disable Control Panel  | Обмежує доступ до Панелі керування та налаштувань Windows              |
| Disable Registry Tools | Блокує доступ до редактора реєстру Windows                             |
| Disable USB            | Забороняє використання знімних USB-накопичувачів                       |
| Audit Policy           | Вмикає аудит вибраних подій безпеки Windows                            |
| Login Script           | Виконує сценарій входу під час автентифікації користувача              |

Ці політики демонструють централізоване адміністрування, посилення безпеки робочих станцій та керування користувацьким середовищем за допомогою Active Directory.

---

## Підключення мережевих дисків

Мережеві диски автоматично підключалися для користувачів домену під час процесу входу в систему.

Приклад мережевого шляху:

```text id="2d5v4q"
\\DC01\Public
```

Підключення диска забезпечує користувачам миттєвий доступ до спільних ресурсів без необхідності ручного налаштування мережевих підключень.

---

## Сценарій входу (Logon Script)

За допомогою Group Policy був розгорнутий сценарій входу для автоматизації налаштування користувачів.

Приклад сценарію входу:

```bat id="7e9m1k"
net use P: \\DC01\Public
```

Сценарій автоматично підключає спільну мережеву папку щоразу, коли користувач домену входить у систему.

---

## Адміністративне призначення

Налаштовані об’єкти Group Policy демонструють поширені адміністративні, безпекові та конфігураційні практики, які використовуються в корпоративних середовищах Windows.

Реалізовані політики забезпечують такі можливості:

* Стандартизація вигляду робочого столу за допомогою корпоративних шпалер
* Захист системних ресурсів шляхом приховування системного диска
* Автоматичне підключення мережевих дисків
* Обмеження доступу до адміністративних інструментів, таких як Command Prompt і Registry Editor
* Запобігання несанкціонованим змінам системної конфігурації
* Контроль використання знімних USB-накопичувачів
* Збір подій безпеки за допомогою Windows Auditing
* Автоматизація налаштування користувачів через сценарій входу

Ці політики підвищують безпеку, спрощують адміністрування та забезпечують однакову конфігурацію робочих станцій у домені.

---

## Обробка Group Policy

Після приєднання до домену Active Directory та входу за допомогою доменного облікового запису клієнти Windows автоматично отримують налаштовані об’єкти Group Policy.

Політики застосовуються під час:

* Запуску комп’ютера
* Входу користувача в систему
* Фоновому оновленні політик
* Ручному оновленні за допомогою `gpupdate`

## Підсумок

Реалізація Group Policy демонструє централізоване адміністрування робочих станцій Windows у середовищі Active Directory.

Розгорнуті політики автоматизують налаштування користувачів, підвищують безпеку кінцевих пристроїв, обмежують несанкціоновані адміністративні дії та спрощують доступ до спільних ресурсів.

Ця реалізація відповідає типовим практикам корпоративного адміністрування, які використовуються для стандартизації клієнтських комп’ютерів, підвищення безпеки та спрощення централізованого керування в середовищах Windows Server.
