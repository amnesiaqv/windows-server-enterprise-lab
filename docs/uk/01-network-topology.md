# 🌐 Топологія мережі

## Огляд

У цьому документі описано мережеву архітектуру, реалізовану в лабораторному проєкті **Enterprise Active Directory Lab**. Середовище моделює інфраструктуру невеликого підприємства, у якій один сервер Windows Server забезпечує централізовані мережеві служби для кількох клієнтських комп'ютерів, приєднаних до домену.

Уся лабораторія була розгорнута за допомогою **Oracle VirtualBox** та ізольована в мережі типу **Internal Network**, що дозволяє всім віртуальним машинам взаємодіяти між собою без відкриття служб для мережі хостової операційної системи.

## Компоненти інфраструктури

| Ім'я вузла | Операційна система | Основна роль |
| ---------- | ------------------ | ------------ |
| DC01 | Windows Server 2025 | Контролер домену (DC) |
| CLIENT01 | Windows 11 Pro | Робоча станція домену |
| CLIENT02 | Windows 11 Pro | Робоча станція домену |

Контролер домену забезпечує всі основні служби інфраструктури, необхідні для роботи лабораторії:

* Active Directory Domain Services (AD DS)
* DNS Server
* DHCP Server
* File Server
* Group Policy Management

Клієнтські комп'ютери приєднані до домену Active Directory та отримують централізовану конфігурацію від сервера.

## Конфігурація мережі

| Параметр | Значення |
| -------- | -------- |
| Гіпервізор | Oracle VirtualBox |
| Тип мережі | Internal Network |
| Домен | KUZNIETSOV.local |
| Контролер домену | DC01 |
| IP-адреса сервера | 192.168.10.10 |
| DHCP-діапазон | 192.168.10.100 – 192.168.10.200 |
| Шлюз за замовчуванням | N/A |
| Основний DNS-сервер | 192.168.10.10 |

Клієнти автоматично отримують конфігурацію IPv4 від служби DHCP, що працює на контролері домену.

## Мережеві служби

### Active Directory

Забезпечує централізовану автентифікацію, авторизацію та служби каталогів для всіх користувачів і комп'ютерів, приєднаних до домену.

---

### DNS

DNS-сервер виконує перетворення імен вузлів у межах домену Active Directory та зберігає інтегровані DNS-записи для доменних служб і клієнтських комп'ютерів.

---

### DHCP

DHCP-сервер автоматично призначає IPv4-адреси, DNS-параметри та інші мережеві налаштування клієнтським робочим станціям, усуваючи необхідність ручної конфігурації.

---

### Файлові служби

Контролер домену розміщує спільні папки, захищені дозволами NTFS і Share Permissions. Доступ до ресурсів контролюється за допомогою груп безпеки Active Directory.

---

### Group Policy

Об'єкти групової політики (GPO) застосовуються до користувачів і комп'ютерів домену для централізованого керування параметрами операційної системи та автоматизації адміністративних завдань.

---

## Перевірка мережі

Для підтвердження коректної роботи мережі були виконані такі перевірки.

| Перевірка | Очікуваний результат | Статус |
| ---------- | -------------------- | :----: |
| Приєднання до домену | Клієнт успішно приєднаний до домену | ✅ |
| Отримання DHCP-оренди | Клієнт автоматично отримує IP-адресу | ✅ |
| DNS-резолюція | Імена вузлів успішно розпізнаються | ✅ |
| ICMP-з'єднання | Клієнти успішно взаємодіють із сервером | ✅ |
| Автентифікація Active Directory | Вхід до домену успішний | ✅ |
| Застосування Group Policy | Політики успішно застосовані | ✅ |
| Доступ до спільних папок | Мережеві ресурси доступні | ✅ |

## Команди перевірки

Під час розгортання та тестування використовувалися такі команди.

### Відображення мережевої конфігурації

```cmd
ipconfig /all
```

### Перевірка мережевого з'єднання

```cmd
ping DC01
```

### Перевірка DNS-резолюції

```cmd
nslookup DC01
```

### Перевірка застосованих групових політик

```cmd
gpresult /r
```

## Підсумок

Реалізована топологія мережі забезпечує централізовану інфраструктуру Windows Server, що підтримує автентифікацію, DNS-резолюцію, автоматичне призначення IP-адрес за допомогою DHCP, керування груповими політиками та безпечний спільний доступ до файлів.

Створене середовище максимально наближене до архітектури, яка широко використовується в корпоративних мережах малого та середнього бізнесу, і є практичною платформою для вивчення адміністрування Windows Server, корпоративних мереж і керування Active Directory з використанням стандартних технологій Microsoft.
