# 📁 Настройка файлового сервера

## Обзор

В этом документе описывается настройка роли **File Server** в рамках лабораторного проекта **Enterprise Active Directory Lab**.

Контроллер домена предоставляет централизованные общие папки, доступные авторизованным пользователям домена. Управление доступом к общим ресурсам осуществляется с помощью комбинации **разрешений общего доступа (Share Permissions)**, **разрешений NTFS** и **групп безопасности Active Directory**.

---

## Файловый сервер

Контроллер домена под управлением **Windows Server 2025** также выполняет роль файлового сервера лабораторной среды.

| Параметр | Значение |
| -------- | -------- |
| Сервер | DC01 |
| Операционная система | Windows Server 2025 |
| Файловая система | NTFS |
| Способ доступа | SMB File Sharing |

Общие папки обеспечивают централизованное хранение данных для пользователей, подключенных к домену Active Directory.

---

## Общие папки

Для имитации ресурсов различных подразделений были созданы несколько общих папок.

| Общий ресурс | Назначение |
| ------------ | ---------- |
| Public | Общие данные компании |
| IT | Файлы ИТ-отдела |
| HR | Документы отдела кадров |
| Finance | Документы финансового отдела |

---

## Разрешения общего доступа (Share Permissions)

Разрешения общего доступа управляют доступом к папкам по сети, тогда как разрешения NTFS определяют права доступа к файлам и каталогам в локальной файловой системе.

Разрешения назначались с использованием **групп безопасности Active Directory**, а не отдельным пользователям.

Такой подход упрощает администрирование и соответствует рекомендациям Microsoft.

---

## Разрешения NTFS

Разрешения NTFS обеспечивают безопасность файловой системы и контролируют доступ к файлам и папкам, хранящимся на сервере.

Разрешения NTFS в основном назначались **группам безопасности Active Directory**, а не отдельным учетным записям пользователей, что соответствует рекомендациям Microsoft по построению масштабируемой модели управления доступом.

Использование групп безопасности значительно упрощает администрирование разрешений и облегчает дальнейшее сопровождение инфраструктуры.

---

## Управление доступом

Доступ к общим ресурсам организован в соответствии с **принципом наименьших привилегий (Least Privilege)**.

Пользователям предоставляются только те права, которые необходимы для выполнения их рабочих задач.

Примеры уровней доступа:

* Чтение (Read)
* Изменение (Modify)
* Полный доступ (Full Control)

Итоговые эффективные разрешения определяются комбинацией **Share Permissions** и **NTFS Permissions**.

---

## Подключение сетевого диска

Для пользователей домена сетевой диск автоматически подключается с помощью **групповой политики (Group Policy)** или **сценария входа (Logon Script)**.

Это позволяет пользователям получать доступ к общим папкам без необходимости вручную подключать сетевые ресурсы.

Пример:

```text
\\DC01\Public
```

## Итоги

Файловый сервер обеспечивает централизованное хранение данных в лабораторном проекте **Enterprise Active Directory Lab**.

Использование **SMB-общих папок**, **разрешений NTFS**, **разрешений общего доступа (Share Permissions)** и **групп безопасности Active Directory** демонстрирует безопасную и масштабируемую модель управления доступом к файлам, широко применяемую в корпоративных инфраструктурах на базе Windows Server.
