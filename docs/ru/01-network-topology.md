# 🌐 Топология сети

## Обзор

В этом документе описана сетевая архитектура, реализованная в лабораторном проекте **Enterprise Active Directory Lab**. Среда имитирует инфраструктуру небольшой компании, в которой один сервер Windows предоставляет централизованные сетевые службы нескольким клиентским компьютерам, входящим в домен.

Вся лаборатория была развернута с использованием **Oracle VirtualBox** и изолирована во внутренней сети (**Internal Network**), что позволяет всем виртуальным машинам взаимодействовать друг с другом без предоставления доступа к службам из сети хостовой операционной системы.

## Компоненты инфраструктуры

| Имя узла | Операционная система | Основная роль |
| -------- | -------------------- | ------------- |
| DC01 | Windows Server 2025 | Контроллер домена (DC) |
| CLIENT01 | Windows 11 Pro | Рабочая станция домена |
| CLIENT02 | Windows 11 Pro | Рабочая станция домена |

Контроллер домена предоставляет все основные службы, необходимые для работы лабораторной среды:

* Active Directory Domain Services (AD DS)
* DNS-сервер
* DHCP-сервер
* Файловый сервер
* Управление групповыми политиками (Group Policy)

Клиентские компьютеры входят в домен Active Directory и получают централизованную конфигурацию от сервера.

---

## Конфигурация сети

| Параметр | Значение |
| -------- | -------- |
| Гипервизор | Oracle VirtualBox |
| Тип сети | Internal Network |
| Домен | KUZNIETSOV.local |
| Контроллер домена | DC01 |
| IP-адрес сервера | 192.168.10.10 |
| Диапазон DHCP | 192.168.10.100 – 192.168.10.200 |
| Шлюз по умолчанию | N/A |
| Предпочитаемый DNS-сервер | 192.168.10.10 |

Клиентские компьютеры автоматически получают параметры IPv4 от службы DHCP, работающей на контроллере домена.

---

## Сетевые службы

### Active Directory

Обеспечивает централизованную аутентификацию, авторизацию и службы каталогов для всех пользователей и компьютеров, входящих в домен.

---

### DNS

DNS-сервер выполняет разрешение доменных имен внутри домена Active Directory и хранит интегрированные DNS-записи, необходимые для работы доменных служб и клиентских компьютеров.

---

### DHCP

DHCP-сервер автоматически назначает клиентским рабочим станциям IPv4-адреса, параметры DNS и другие сетевые настройки, исключая необходимость ручной конфигурации.

---

### Файловые службы

Контроллер домена предоставляет общие папки, защищенные разрешениями NTFS и разрешениями общего доступа (Share Permissions). Управление доступом осуществляется с помощью групп безопасности Active Directory.

---

### Групповые политики

Объекты групповой политики (Group Policy Objects, GPO) применяются к пользователям и компьютерам домена для централизованного управления настройками операционной системы и автоматизации административных задач.

---

## Проверка работоспособности сети

Для подтверждения корректной работы сети были выполнены следующие проверки.

| Проверка | Ожидаемый результат | Статус |
| -------- | ------------------- | :----: |
| Присоединение к домену | Клиент успешно добавлен в домен | ✅ |
| Получение аренды DHCP | Клиент автоматически получает IP-адрес | ✅ |
| Разрешение DNS-имен | Имена узлов корректно разрешаются | ✅ |
| Проверка ICMP-соединения | Клиенты успешно взаимодействуют с сервером | ✅ |
| Аутентификация Active Directory | Вход в домен выполняется успешно | ✅ |
| Применение групповых политик | Политики успешно применяются | ✅ |
| Доступ к общим папкам | Сетевые ресурсы доступны | ✅ |

## Команды проверки

Во время развертывания и тестирования использовались следующие команды.

### Просмотр сетевой конфигурации

```cmd
ipconfig /all
```

### Проверка сетевого соединения

```cmd
ping DC01
```

### Проверка разрешения DNS

```cmd
nslookup DC01
```

### Проверка применённых групповых политик

```cmd
gpresult /r
```

## Итоги

Реализованная топология сети обеспечивает централизованную инфраструктуру Windows Server, включающую аутентификацию пользователей, разрешение DNS-имен, автоматическое назначение IP-адресов через DHCP, управление групповыми политиками и безопасный общий доступ к файлам.

Созданная лабораторная среда максимально приближена к архитектуре, широко используемой в корпоративных сетях малого и среднего бизнеса, и служит практической платформой для изучения администрирования Windows Server, корпоративных сетевых технологий и управления инфраструктурой Active Directory с использованием стандартных технологий Microsoft.
