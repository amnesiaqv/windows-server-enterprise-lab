# 📡 Настройка DHCP-сервера

## Обзор

В этом документе описывается развертывание и настройка службы **Dynamic Host Configuration Protocol (DHCP)** в рамках лабораторного проекта **Enterprise Active Directory Lab**.

DHCP-сервер автоматически назначает клиентским компьютерам, входящим в домен, IPv4-адреса, параметры DNS и другие сетевые настройки, упрощая администрирование сети и обеспечивая единообразную конфигурацию.

---

## Роль DHCP-сервера

Роль **DHCP Server** была установлена на контроллер домена **DC01**.

| Параметр | Значение |
| -------- | -------- |
| DHCP-сервер | DC01 |
| Операционная система | Windows Server 2025 |
| Авторизация | Авторизован в Active Directory |
| Семейство адресов | IPv4 |

Перед началом выдачи IP-адресов клиентам DHCP-сервер был авторизован в Active Directory.

---

## Область DHCP

Для внутренней лабораторной сети была настроена одна область (Scope) IPv4.

| Параметр | Значение |
| -------- | -------- |
| Имя области | Internal Network |
| Сеть | 192.168.10.0/24 |
| Начальный адрес | 192.168.10.100 |
| Конечный адрес | 192.168.10.200 |
| Маска подсети | 255.255.255.0 |
| Шлюз по умолчанию | N/A |
| Предпочитаемый DNS-сервер | 192.168.10.10 |

Данная область обеспечивает автоматическое назначение IP-адресов всем клиентским компьютерам под управлением Windows 11.

---

## Параметры DHCP

Были настроены следующие параметры DHCP.

| Параметр | Значение |
| -------- | -------- |
| DNS-сервер (006) | 192.168.10.10 |
| DNS-домен (015) | KUZNIETSOV.local |

Эти параметры гарантируют, что во время получения аренды DHCP клиенты автоматически получают необходимые сетевые настройки.

---

## Выданные аренды

После присоединения к домену и обновления сетевой конфигурации оба клиентских компьютера успешно получили IP-адреса от DHCP-сервера.

| Компьютер | Источник адреса |
| --------- | --------------- |
| CLIENT01 | DHCP |
| CLIENT02 | DHCP |

Активные аренды отображаются в консоли управления DHCP в разделе **Address Leases**.

---

## Процесс получения аренды DHCP

Каждый клиент проходит стандартный процесс получения аренды DHCP:

1. DHCP Discover
2. DHCP Offer
3. DHCP Request
4. DHCP Acknowledgement (ACK)

После получения аренды клиент автоматически получает:

* IPv4-адрес
* Маску подсети
* Предпочитаемый DNS-сервер
* DNS-суффикс

## Команды проверки

Во время тестирования использовались следующие команды.

### Просмотр сетевой конфигурации клиента

```cmd
ipconfig /all
```

### Обновление аренды DHCP

```cmd
ipconfig /renew
```

### Освобождение текущей аренды

```cmd
ipconfig /release
```

## Итоги

DHCP-сервер успешно обеспечивает автоматическую настройку IPv4 для всех клиентских компьютеров Windows 11, входящих в домен.

Централизованное управление адресным пространством снижает административную нагрузку, минимизирует вероятность ошибок конфигурации и обеспечивает единообразные сетевые настройки во всей лабораторной среде. Все клиенты успешно получили действующие DHCP-аренды, параметры DNS и конфигурацию домена от авторизованного DHCP-сервера.
