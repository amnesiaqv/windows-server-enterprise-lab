# 🌐 Настройка DNS-сервера

## Обзор

В этом документе описывается развертывание и настройка службы **Domain Name System (DNS)** в рамках лабораторного проекта **Enterprise Active Directory Lab**.

DNS-сервер интегрирован со **службами доменов Active Directory (Active Directory Domain Services, AD DS)** и обеспечивает разрешение имен для всех компьютеров, входящих в домен. Это позволяет клиентам находить контроллер домена и другие сетевые ресурсы по имени хоста, а не по IP-адресу.

---

## Роль DNS-сервера

Роль **DNS Server** была установлена на контроллер домена **DC01** во время развертывания Active Directory.

| Параметр | Значение |
| -------- | -------- |
| DNS-сервер | DC01 |
| Операционная система | Windows Server 2025 |
| Тип DNS-зоны | Интегрированная с Active Directory |
| Имя домена | KUZNIETSOV.local |

Служба DNS хранит DNS-записи в Active Directory и автоматически реплицирует их между контроллерами домена, обеспечивая безопасное и централизованное разрешение имен.

---

## Настройка DNS-зоны

Во время установки Active Directory была автоматически создана **зона прямого просмотра (Forward Lookup Zone)**.

| Зона | Тип |
| ---- | --- |
| KUZNIETSOV.local | Зона прямого просмотра (Forward Lookup Zone) |

Во время повышения сервера до роли контроллера домена зона была автоматически настроена как интегрированная с Active Directory.

---

## Записи узлов

В DNS-зоне были успешно зарегистрированы следующие записи типа **Host (A)**.

| Имя узла | Назначение |
| -------- | ---------- |
| DC01 | Контроллер домена |
| CLIENT01 | Рабочая станция домена |
| CLIENT02 | Рабочая станция домена |

Динамические обновления DNS позволяют клиентским компьютерам автоматически регистрировать и обновлять свои DNS-записи после присоединения к домену.

---

## Процесс разрешения имен DNS

DNS-служба выполняет следующие функции:

* Преобразование имен узлов в IP-адреса
* Поддержка аутентификации Active Directory
* Поиск контроллеров домена
* Обеспечение взаимодействия между участниками домена
* Поддержка динамических DNS-записей

Без правильно настроенного DNS-сервера вход в домен и работа многих служб Active Directory были бы невозможны.

## Команды проверки

Для проверки работоспособности DNS использовались следующие команды.

### Проверка разрешения имени контроллера домена

```cmd
nslookup DC01
```

### Проверка разрешения имен клиентских компьютеров

```cmd
nslookup CLIENT01
```

```cmd
nslookup CLIENT02
```

### Просмотр сетевой конфигурации

```cmd
ipconfig /all
```

На каждом клиентском компьютере в качестве предпочитаемого DNS-сервера указан контроллер домена.
