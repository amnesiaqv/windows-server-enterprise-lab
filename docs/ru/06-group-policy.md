# 🛡️ Настройка групповых политик (Group Policy)

## Обзор

В этом документе описывается настройка и развертывание объектов групповой политики (Group Policy Objects, GPO) в лабораторном проекте **Enterprise Active Directory Lab**.

Group Policy — одна из ключевых технологий централизованного управления в Microsoft Active Directory. Она позволяет администраторам централизованно настраивать параметры операционной системы, применять политики безопасности, автоматизировать административные задачи и обеспечивать единообразную конфигурацию пользовательской среды на всех компьютерах, входящих в домен.

---

## Управление групповыми политиками

Объекты групповой политики были созданы и настроены с помощью **Group Policy Management Console (GPMC)**.

| Компонент | Значение |
| ---------- | -------- |
| Инструмент управления | Group Policy Management |
| Домен | KUZNIETSOV.local |
| Контроллер домена | DC01 |
| Клиентская ОС | Windows 11 Pro |

Политики были привязаны к соответствующим организационным подразделениям (OU), чтобы применяться только к нужным пользователям или компьютерам.

---

## Реализованные объекты групповой политики

В лабораторной среде были успешно настроены следующие объекты групповой политики.

| Групповая политика | Назначение |
| ------------------ | ---------- |
| Set Wallpaper | Устанавливает корпоративные обои рабочего стола для пользователей домена |
| Hide Local C: Drive | Скрывает локальный диск C: в Проводнике Windows |
| Map Network Drives | Автоматически подключает сетевые диски при входе пользователя |
| Disable CMD | Запрещает запуск командной строки |
| Disable Control Panel | Ограничивает доступ к Панели управления и приложению «Параметры» |
| Disable Registry Tools | Блокирует доступ к редактору реестра Windows |
| Disable USB | Запрещает использование съемных USB-накопителей |
| Audit Policy | Включает аудит выбранных событий безопасности Windows |
| Login Script | Выполняет сценарий входа пользователя при аутентификации |

Эти политики демонстрируют возможности централизованного администрирования, повышения безопасности рабочих станций и управления пользовательской средой с помощью Active Directory.

---

## Подключение сетевых дисков

Сетевые диски автоматически подключались для пользователей домена во время входа в систему.

Пример сетевого пути:

```text
\\DC01\Public
```

Автоматическое подключение позволяет пользователям сразу получать доступ к общим ресурсам без необходимости ручной настройки.

---

## Сценарий входа

Сценарий входа был развернут с помощью Group Policy для автоматизации пользовательской конфигурации.

Пример сценария:

```bat
net use P: \\DC01\Public
```

Сценарий автоматически подключает общую сетевую папку каждый раз при входе пользователя в домен.

---

## Назначение политик

Настроенные объекты групповой политики демонстрируют распространённые методы администрирования, обеспечения безопасности и централизованного управления конфигурацией, применяемые в корпоративных средах Windows.

Реализованные политики обеспечивают следующие возможности:

* Единообразное оформление рабочего стола с использованием корпоративных обоев
* Защиту системных ресурсов путем скрытия системного диска
* Автоматическое подключение сетевых дисков
* Ограничение доступа к административным инструментам, таким как Командная строка и Редактор реестра
* Предотвращение несанкционированного изменения конфигурации системы
* Контроль использования съемных USB-накопителей
* Сбор событий безопасности с помощью механизма аудита Windows
* Автоматическую настройку пользовательской среды с помощью сценария входа

Эти политики повышают уровень безопасности, упрощают администрирование и обеспечивают единообразную конфигурацию рабочих станций во всем домене.

---

## Обработка групповых политик

После присоединения компьютера к домену Active Directory и входа пользователя с доменной учетной записью клиенты Windows автоматически получают настроенные объекты групповой политики.

Политики применяются во время:

* запуска компьютера;
* входа пользователя в систему;
* фонового обновления политик;
* ручного обновления с помощью команды `gpupdate`.

```cmd

```

## Итоги

Реализация Group Policy демонстрирует возможности централизованного администрирования рабочих станций Windows в среде Active Directory.

Настроенные политики автоматизируют конфигурацию пользовательской среды, повышают безопасность конечных устройств, ограничивают выполнение несанкционированных административных действий и упрощают доступ к общим сетевым ресурсам.

Данная реализация отражает распространённые практики корпоративного администрирования, используемые для стандартизации клиентских компьютеров, повышения безопасности рабочих станций и упрощения централизованного управления в инфраструктуре Windows Server.
