# 🏛️ Службы доменов Active Directory

### Обзор

В этом документе описывается развертывание и настройка **Active Directory Domain Services (AD DS)** в рамках лабораторного проекта **Enterprise Active Directory Lab**.

Active Directory является центральной платформой управления учетными записями и доступом в лабораторной среде. Она обеспечивает централизованную аутентификацию, авторизацию, управление компьютерами и развертывание групповых политик для всех устройств, входящих в домен.

---

## Развертывание Active Directory

Роль **Active Directory Domain Services (AD DS)** была установлена на виртуальную машину с **Windows Server 2025 (DC01)**.

В процессе развертывания сервер был повышен до роли контроллера домена (Domain Controller), в результате чего был создан новый лес и домен Active Directory.

| Параметр | Значение |
| -------- | -------- |
| Контроллер домена | DC01 |
| Операционная система | Windows Server 2025 |
| Имя домена | KUZNIETSOV.local |
| Функциональный уровень леса | Windows Server 2025 (по умолчанию) |
| Функциональный уровень домена | Windows Server 2025 (по умолчанию) |

---

## Структура организационных единиц

Для логического разделения административных объектов и упрощения управления были созданы **организационные единицы (Organizational Units, OU)**.

Пример структуры:

```text
KUZNIETSOV.local
│
└── Company
    ├── Users
    │   ├── IT
    │   ├── HR
    │   └── Finance
    │
    ├── Groups
    ├── Computers
    └── Servers
```

Разделение ресурсов по отдельным организационным единицам упрощает делегирование административных полномочий и применение объектов групповой политики (GPO) к определенным подразделениям или типам объектов.

---

## Учетные записи пользователей

Для проверки централизованной аутентификации и управления доступом были созданы доменные учетные записи пользователей.

Каждая учетная запись была размещена в соответствующей организационной единице и включена в одну или несколько групп безопасности в соответствии с требованиями подразделения.

Примеры атрибутов пользователя:

* Имя пользователя (Username)
* Отображаемое имя (Display Name)
* Подразделение (Department)
* Пароль
* Членство в группах безопасности

---

## Группы безопасности

Для упрощения управления разрешениями были созданы группы безопасности.

Вместо назначения разрешений отдельным пользователям права доступа предоставляются группам, а пользователи получают необходимые разрешения через членство в этих группах.

Примеры групп подразделений:

* IT
* HR
* Finance

Такой подход соответствует рекомендуемой Microsoft модели управления доступом на основе ролей (**Role-Based Access Control, RBAC**).

## Аутентификация

Аутентификация пользователей домена выполняется контроллером домена.

Основные преимущества:

* Централизованное управление учетными записями
* Единый вход (Single Sign-On, SSO)
* Применение политик паролей
* Аутентификация на основе протокола Kerberos
* Централизованное администрирование

Пользователям больше не требуется создавать отдельные локальные учетные записи на каждой рабочей станции.

## Итоги

Развертывание Active Directory обеспечивает централизованное управление учетными записями и доступом в лабораторной среде.

Использование организационных единиц (OU), доменных пользователей, групп безопасности и рабочих станций Windows 11, присоединенных к домену, демонстрирует основные принципы управления корпоративной инфраструктурой и административные практики, широко применяемые в производственных средах на базе Windows Server.
